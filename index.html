01 What is a Symbol Table?

A Symbol Table is a data structure used by a compiler to store information about every identifier — variables, functions, and arrays — found in source code. It records each identifier's name, type, scope, and memory location. Without it, the compiler cannot verify logical correctness. It is the backbone of semantic analysis.

02 The 5-Step Pipeline

The symbol table is built through a five-step pipeline: source code is tokenized, parsed for declarations, identifiers are stored with their attributes, the table is displayed, and finally used for semantic validation. Each step feeds the next.

03 Scope Management

Scope defines where an identifier is valid. The compiler enters a new scope on { and exits on }. Three levels exist: global, function, and block. Accessing a variable outside its scope causes a lookup failure and a compiler error.

04 Semantic Analysis and Errors

Once built, the symbol table is used to catch four types of semantic errors: undeclared variables, redeclarations, type mismatches, and scope violations. These are invisible to a syntax checker but immediately detectable through symbol table lookups.

Error Types
Error TypeCauseExampleHow DetectedFix
Undeclared Variable Used before declared z = 5; lookup("z") returns null Declare z first
Redeclaration Same name in same scope int x; int x; insert("x") finds duplicate Remove duplicate
Type Mismatch Wrong type assigned int x = "hello"; Stored type int ≠ string Match types or cast
Scope Violation Access outside scope use sum; // global lookup("sum", global) → null Move access inside add()
